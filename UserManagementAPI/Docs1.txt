Documentación del Proyecto: UserManagementAPI
Descripción
Este proyecto es una API RESTful construida con ASP.NET Core para gestionar usuarios. Permite crear, actualizar, recuperar y eliminar registros de usuarios de manera eficiente.

Estructura Principal
Modelo de Usuario:
Se define la clase pública Usuario con las propiedades Id, Nombre y Email.

Almacenamiento:
Se utiliza una lista estática usuarios para almacenar los usuarios en memoria (ideal para pruebas y desarrollo).

Endpoints:
Se implementan los siguientes endpoints usando Minimal APIs:

GET /users: Recupera todos los usuarios o una lista específica por IDs usando la query string ids.
GET /users/{id}: Recupera un usuario por su ID.
POST /users/: Agrega un nuevo usuario.
PUT /users/{id}: Actualiza los datos de un usuario existente.
DELETE /users/{id}: Elimina un usuario por su ID.
Detalles Técnicos
Accesibilidad:
La clase Usuario es pública para evitar problemas de acceso desde otras partes del proyecto.
Métodos Estáticos:
Los métodos que gestionan los usuarios son estáticos para que puedan ser referenciados directamente en los endpoints.
Enlace de Parámetros:
Los endpoints están diseñados para recibir parámetros correctamente desde la ruta y el cuerpo de la petición, siguiendo las convenciones de Minimal APIs.
Ejemplo de Uso
Recuperar usuarios específicos:
GET /users?ids=1&ids=2
Agregar usuario:
POST /users/ con un cuerpo JSON:
Actualizar usuario:
PUT /users/3 con un cuerpo JSON actualizado.
Eliminar usuario:
DELETE /users/3
Consideraciones
El almacenamiento es en memoria, por lo que los datos se pierden al reiniciar la aplicación.
Para producción, se recomienda usar una base de datos y separar el modelo en archivos independientes.
